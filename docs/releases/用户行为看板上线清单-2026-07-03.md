## 上线前必须准备

### 1. 申请 `LOKI_TOKEN`

- 配置项：`grafana-loki.token`，环境变量 `LOKI_TOKEN`，`application-prod.yml` 里默认空字符串兜底，**不设置的话看板会报 `BEHAVIOR_LOKI_TOKEN_MISSING`**

  - `LOKI_BASE_URL`（默认 `https://logs.hiqlcd.com`）
  - `LOKI_DATASOURCE_UID`（默认 `ee8hccr0l3nr4a`）
  - `LOKI_DATASOURCE_ID`（默认 `1`）

### 2. 手动执行迁移 SQL（本项目未接入 Flyway，必须手动跑）

```sql
-- ============================================================
-- V1.2.17 用户行为看板：菜单 + 权限
--
-- 作用：在大后台新增一级菜单「用户行为看板」，并配套权限标识，
--       授予 super_admin 角色；其余角色请在【权限管理】中分配。
--
-- 重要约定：
--   1. ts_menu.component 必须等于前端路由 name = 'UserBehaviorDashboard'
--      （前端 pickRoutes 通过 component === route.name 匹配后才注册路由、显示菜单）。
--   2. 权限标识 = 'system:behavior-dashboard:query'，与
--      SystemUserBehaviorAdminController 上的 @SaCheckPermission 一致。
--      注意：本项目 super_admin 无权限通配符，必须显式授予，否则连超管也会被拦截。
--
-- 执行库：hiq_admin（PostgreSQL）。本项目未启用 Flyway，请手动执行：
--   psql "$ADMIN_DB_URL" -f V1.2.17__add_behavior_dashboard_menu.sql
--
-- 幂等：可重复执行（NOT EXISTS 保护）。
-- ============================================================

DO $$
DECLARE
    v_menu_id    varchar := 'b1a2c3d4-0000-0000-0000-0000000d0001';
    v_product_id varchar;
    v_role_id    varchar;
BEGIN
    -- 解析产品线 id：与大后台前端产品编码对应。
    -- 生产环境 ts_product 里大后台的 code 实际是 backend_admin。
    SELECT id INTO v_product_id
      FROM ts_product
     WHERE code = 'backend_admin' AND is_deleted = FALSE
     LIMIT 1;

    -- 1. 一级菜单（同时承载侧边栏显示 + 查询权限标识）
    IF NOT EXISTS (SELECT 1 FROM ts_menu WHERE id = v_menu_id) THEN
        INSERT INTO ts_menu
            (id, name, code, permission, type, sort, parent_id, path, icon,
             component, component_name, status, visible, always_show, title,
             product_id, create_time, update_time, is_deleted)
        VALUES
            (v_menu_id, '用户行为看板', 'behavior_dashboard', 'system:behavior-dashboard:query',
             'MENU', 100, NULL, '/dashboard/behavior', 'BarChartOutlined',
             'UserBehaviorDashboard', 'UserBehaviorDashboard', 'ENABLED', TRUE, TRUE, '用户行为看板',
             v_product_id, now(), now(), FALSE);
    END IF;

    -- 2. 授权给 super_admin（保证超管可见可访问）
    SELECT id INTO v_role_id
      FROM ts_role
     WHERE code = 'super_admin' AND is_deleted = FALSE
     LIMIT 1;

    IF v_role_id IS NOT NULL
       AND NOT EXISTS (
           SELECT 1 FROM ts_role_menu
            WHERE role_id = v_role_id AND menu_id = v_menu_id AND is_deleted = FALSE
       ) THEN
        INSERT INTO ts_role_menu (id, role_id, menu_id, create_time, update_time, is_deleted)
        VALUES (gen_random_uuid()::varchar, v_role_id, v_menu_id, now(), now(), FALSE);
    END IF;
END $$;

-- 验证：
--   SELECT id, name, component, permission, type, status FROM ts_menu WHERE id = 'b1a2c3d4-0000-0000-0000-0000000d0001';
--   SELECT rm.* FROM ts_role_menu rm JOIN ts_role r ON r.id = rm.role_id
--     WHERE rm.menu_id = 'b1a2c3d4-0000-0000-0000-0000000d0001';
--
-- 给其它角色授权（示例，将 <role_code> 替换为目标角色编码）：
--   INSERT INTO ts_role_menu (id, role_id, menu_id, create_time, update_time, is_deleted)
--   SELECT gen_random_uuid()::varchar, r.id, 'b1a2c3d4-0000-0000-0000-0000000d0001', now(), now(), FALSE
--     FROM ts_role r WHERE r.code = '<role_code>' AND r.is_deleted = FALSE;
```

### 3. 确认权限覆盖范围

SQL 默认只授权给 `super_admin`。**先确认清楚哪些角色需要看这个看板**（运营/产品/数据团队等），逐个跑上面"给其它角色授权"的示例语句。不跑的话，非 super_admin 账号即使菜单在库里也看不到（fail-closed，符合预期，但业务上要提前想清楚给谁开）。
