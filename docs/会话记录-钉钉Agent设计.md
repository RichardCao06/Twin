## 5. DWS-Agent 方案设计（workflow）

需求：设计一个连接钉钉、代理用户日常工作、且能持续进化的 Agent，需覆盖架构、上下文管理、钉钉历史知识管理、质量保障、危险操作防护、迭代优化。

执行方式：用 **workflow** 跑了 9 个 agent —— 6 维度并行设计 → 综合 → 对抗式审查（揪出 5 个 high / 共 14 个问题）→ 吸收修复后定稿。

**核心设计要点**：
- **铁律**：会思考的 Worker（Claude 子会话）不可信，只产出意图 JSON；真正调 dws 的是无 LLM 的确定性 Executor。
- **三道纵深防御**：① 执行权与 Worker 解耦（OS 权限 + token，非 PATH）；② PolicyGate 按 argv 判级 + default-deny（不依赖 `--yes`）；③ 可验证性矩阵替代 dry-run（前置只读查 + 执行后真实回查）。
- **隐私**：Keychain 加密落盘、字段最小化、分级脱敏、污点传播 block、单聊硬过滤。
- **质量**：确定性回查替代 LLM 自评、eval 金标准来自人工 correction、高影响域全审。
- **进化**：只落数据不落代码、playbook 防固化、能力探测可恢复。
- **路线图**：阶段 0 地基 → 1 MVP(只读+全人工确认) → 2 低风险 R1 自治 → 3 受控自治（R2/R3 永远人确认，无全自动终态），每阶段有准入/退出门槛。

完整方案见 `dws-agent-设计方案.md`。

## 6. 认证路线确认

- 现状：走**个人 OAuth 扫码登录**（用户态），凭证存本机 `~/.dws/`；access_token 2h、refresh_token 30 天且使用后轮转。
- 关键结论：「单设备独占」的真实含义是"是否存在第二个独立授权各自刷新"，而非进程数量。
- **定下路线**：Agent 仅本机单实例运行 + **共享同一份 `~/.dws` 凭证** + **刷新文件锁串行化**，即可安全使用 refresh_token 方案，无需应用级凭证。
- 已写入方案 §4，并在阶段 0 脚手架新增"共享凭证 + 刷新文件锁"工作项与并发 24h 不掉线的退出条件。

## 7. 待定项拍板与方案定稿

剩余 9 项待定全部按推荐方案确定，写入方案 §0「已确定决策」：

| # | 决策项 | 采纳方案 |
|---|---|---|
| 1 | 落盘加密与保留期 | Keychain 文件级加密；实体卡只存 ID+名；raw 90 天 TTL |
| 2 | 认证路线 | 本机单实例 + 共享凭证 + 刷新文件锁 |
| 3 | KB 群白名单/敏感度 | 阶段 1 自动探测后逐群点选；未标群默认 high |
| 4 | scope 初始化 | 阶段 1 自动探测后确认 |
| 5 | 向量检索 | MVP 用 FTS 中文 bigram 降级，后续再上向量 |
| 6 | dwsd 常驻方式 | macOS launchd KeepAlive |
| 7 | 自治优先序 | 催日报/会前提醒 → 审批拟办 → 群通知；审批决策与移除成员永久人确认 |
| 8 | Executor 隔离 | 阶段 0 单层 shim+token；阶段 1 退出前升级独立账号+chmod 750 |
| 9 | SKILL 自动改写 | 只产 diff 不自动 apply，禁碰危险段 |

至此方案无待定项，可进入阶段 0 脚手架开发。
